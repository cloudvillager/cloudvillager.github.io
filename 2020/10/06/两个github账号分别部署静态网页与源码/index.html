两个github账号分别部署静态网页与源码
发表于2020-10-06|更新于2020-10-10|git
|阅读量:

前言

​ 前几天一直在想这个东西，怎么做个没人发现得网站，之前水墨得那个博客其实也可以，但是我有俩学长好像关注了，我不知道我这边这边更新，那边学长会不会看到，而且玩github得几乎也都知道有这个hexo得事情，哪天有人知道了我的github账号，搜一下，我的内心活动全都敞开了，哈哈哈，好吧，其实也没什么，但是就是感觉有人看着，心里面不好受，然后就重新注册了一个github账号，而且之前的next主题是越看越丑，黑白….就重新搭建了一个，用的butterfly主题，在本地搭建的又一小部分了，就一直在想这个东西怎么搞，嘿嘿，现在博客我也不下架了，就在哪里放着吧，哪天更新一下文章就行，这个新博客我要好好经营。。。emm平时内心小活动就在这里写嘿嘿嘿，自己得一方小天地。

​ 好吧，现在来说说这个两个账号咋回事，水墨账号就平时写项目啥得都用那个，上传也上传那个仓库，拾光账号就用来专门为建立没人知道得个人网站而创建得，其他什么也不管，hexo生成得静态网页放到这里，然后源码放到另外一个github账号，私有仓库，很完美哈哈，然后今天下午我就遇到了一个新的问题，就是，当我准备将hexo生成得静态网页新的github时，报错了，现在知道原因了，是因为我之前只配了水墨得ssh公钥与github得连接，然后我就懵逼了。

​ 下面是我总结的

第一次配置git与github

首先回顾普通第一次配置git配置得时候

配置第二个github账号，

  1. 再次生成一个ssh key,

    之前都是直接回车，但是这里出现提示输入文件名的时候Enter file in which to save the key (~/.ssh/id_rsa): id_rsa_new要输入与默认配置不一样的文件名，比如：我这里填的是 id_rsa_new，然后在C盘那个文件下就会生成新的id_rsa_new和id_rsa_new.pub

  2. 执行ssh-agent让ssh识别新的私钥

    因为默认只读取id_rsa，为了让SSH识别新的私钥，需将其添加到SSH agent中

    如果出现Could not open a connection to your authentication agent的错误，就试着用以下命令

  3. 配置~/.ssh/config文件
    然后在这个同样的文件夹下，建立一个名为config文件,也可以用命令touch config

    内容写类似这样

    我自己的文件写的是

路径要写绝对路径，昨天晚上配置的时候后面验证就一直报错，和github连接不上，改成绝对路径就好了

  1. 然后把生成的ssh公钥内容添加到新的github账号中

  2. 测试

分别部署不同账号不同远程库

每次上传对应的远程库，在本地库先设置局部用户名和邮箱，如果不设置默认上传就是之前设置的全局的

所以我最后个人网站静态网页放着另外一个github账户仓库，源码放在本来的github私有仓库
在本地博客站点文件夹

然后中间还要分别输入对应的github账号和密码
